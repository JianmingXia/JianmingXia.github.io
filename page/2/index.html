大型网站技术架构

发表于 2018-02-24 | 分类于 读书笔记

说明

网站都是从小网站一步一步发展为大型网站的，而这之中的挑战主要来自于庞大的用户、安全环境恶劣、高并发的访问和海量的数据，任何简单的业务处理，一旦需要处理数以P计的数据和面对数以亿计的用户时，问题就会变的很棘手。
在当前的项目也做了一年半的时间了，当用户量一点一点增大的时候，项目部署、更新的方式还是像之前一样。晚上十点宕机更新，不得不说，方式很粗暴，碰到一些问题，更新到一两点也是有的。而这些问题原本都可以通过技术手段解决，只不过这一年多的时间，都投入在了“新需求、新功能”上。

2015年终总结

发表于 2018-01-01 | 分类于 总结

参与和主持的项目

  1. 会议系统
    • 项目背景：方便快捷得对公司的会议室进行管理
    • 项目目标：会议系统实现会议室预约，添加会议参与人并可进行提醒，员工也可查看与自己相关的会议
    • 我的角色分工：主持
    • 项目总结：会议系统是实习时参与的项目，当时对相关技术都不是很熟悉，所以当时做的时候遇到一些难题：
      （1）会议系统的整个页面的绘制：会议系统整个页面较一般的网络页面有所不同，包括上方会议室的显示，被预约的会议室在页面中的渲染以及会议可被点击，修改会议内容等；
      （2）对接企业QQ：对接企业QQ的SDK是由老李做的，当时在对接企业QQ这块也算是个新的体验，不过由于这块需求是后面加的，所以在添加的时候还是略显麻烦。
      在这次独立负责会议系统中，特别是会议系统界面，让我明白了没有做不了的需求，即时是10000+行的代码，依然可以修改它，改成我们所需要的效果。

2017年终总结

发表于 2018-01-01 | 分类于 总结

前记

最近在查看文档时，看到自己写的《2015年终总结》，真是时间匆匆啊，转眼已经工作了三年多了。从2014.11.04号开始实习，到今天2018.01.07，我在乐港已经待了三年多了。由于工作需要，写了14和15年的总结，16年没有规定了，所以总结也没有写（惰性真真的是恐怖）。17年过去了，我总想留下点什么，至少，了解到今年到底做了什么，是虚度光阴，还是有所作为呢。

15年

15年毕业之后，正式入职，工作基本上都是内网系统的研发，偶尔会有几个H5活动页或者安卓的SDK，工作的第一年，技术栈很杂。入门级的php，搭配yaf以及前端技术，做了n个企业内部的后台。当效率或者并发处理不足的时候，引入了swoole和C++（由于swoole内存占用的原因，后来又废弃了）。
作为一个校招新人，在工作上总体表现不错，但是在竞选新人时失败，原因自然是临场表现不佳，这也是多年以来的通病了，但是是第一次切身感受到语言表达会限制自身的发展（至少，实际的表现没有被认可）。
